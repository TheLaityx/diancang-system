# 扫码点餐系统 - 技术规格说明书

## 项目概述

**项目名称**: 扫码点餐小程序  
**项目类型**: 微信原生小程序 + 微信云开发  
**核心功能**: 基于餐桌二维码的点餐系统，支持用户扫码点餐、商家订单管理  
**目标用户**: 餐饮门店顾客、商家管理人员

---

## 三、数据库设计

### 3.1 集合 (Collections)

#### tables (桌台表)
```javascript
{
  _id: ObjectId,
  tableNo: Number,        // 桌号 1-10
  qrCode: String,         // 二维码内容
  status: Number,         // 0:空闲 1:使用中
  bindUserId: String,     // 绑定的用户ID
  bindTime: Date,         // 绑定时间
  createTime: Date
}
```

#### categories (分类表)
```javascript
{
  _id: ObjectId,
  name: String,          // 分类名称
  sort: Number,          // 排序
  status: Number,        // 0:禁用 1:启用
  createTime: Date
}
```

#### dishes (菜品表)
```javascript
{
  _id: ObjectId,
  categoryId: String,    // 分类ID
  name: String,           // 菜品名称
  image: String,          // 图片URL
  price: Number,          // 价格(分)
  description: String,    // 描述
  specs: [                // 规格选项
    {
      name: String,       // 规格名
      options: [{          // 选项
        name: String,
        price: Number      // 额外价格(分)
      }]
    }
  ],
  stock: Number,          // 库存 -1:无限
  status: Number,         // 0:下架 1:上架
  sales: Number,          // 销量
  createTime: Date
}
```

#### orders (订单表)
```javascript
{
  _id: ObjectId,
  orderNo: String,        // 订单号
  userId: String,         // 用户ID
  tableNo: Number,         // 桌号
  takeNo: String,         // 取餐号
  items: [                 // 订单项目
    {
      dishId: String,
      name: String,
      image: String,
      price: Number,
      quantity: Number,
      specs: String,
      subtotal: Number
    }
  ],
  totalAmount: Number,     // 总金额(分)
  remark: String,          // 备注
  status: Number,          // 0:待支付 1:待接单 2:制作中 3:已完成 4:退款中 5:已退款 6:已取消
  payTime: Date,           // 支付时间
  cookTime: Date,          // 开始制作时间
  completeTime: Date,      // 完成时间
  createTime: Date,
  updateTime: Date
}
```

#### refunds (退款表)
```javascript
{
  _id: ObjectId,
  orderId: String,         // 订单ID
  orderNo: String,         // 订单号
  userId: String,         // 用户ID
  amount: Number,          // 退款金额(分)
  reason: String,          // 退款原因
  status: Number,          // 0:待审核 1:已退款 2:已拒绝
  rejectReason: String,    // 拒绝原因
  createTime: Date,
  processTime: Date
}
```

#### users (用户表)
```javascript
{
  _id: ObjectId,
  openId: String,         // 微信openId
  nickName: String,        // 昵称
  avatarUrl: String,        // 头像
  phone: String,            // 手机号(可选)
  createTime: Date
}
```

#### merchants (商家表)
```javascript
{
  _id: ObjectId,
  username: String,        // 账号
  password: String,        // 密码(MD5)
  name: String,            // 商家名称
  createTime: Date,
  updateTime: Date
}
```

## 六、交互流程

### 6.1 用户点餐流程

```
1. 扫码 → 识别桌号 → 绑定桌号
     ↓
2. 浏览菜品 → 加入购物车
     ↓
3. 购物车 → 确认订单 → 支付
     ↓
4. 支付成功 → 生成取餐号 → 等待取餐
     ↓
5. 取餐 → 完成用餐
     ↓ (可选)
6. 退款申请 → 等待审核
```

### 6.2 商家操作流程

```
1. 商家登录 → 进入管理后台
     ↓
2. 查看新订单 → 接单/拒绝
     ↓
3. 制作中 → 标记出餐 → 完成
     ↓
4. 审核退款申请
     ↓
5. 菜品管理 → 上架/下架/编辑
     ↓
6. 查看数据统计
```
